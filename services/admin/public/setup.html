Streamerbot einrichten

Verbinde deinen Kanal in 4 Schritten. Alles zum Kopieren oder als Download-Paket.

Das .zip enthält alle C#-Actions, die WebSocket-Client-Werte und diese Anleitung als Textdatei. Hinweis: Streamerbot importiert kein natives Ein-Klick-Paket von uns (das interne Format ist undokumentiert) — die Actions fügst du als Execute C# Code ein (Copy-Buttons unten).

1 WebSocket-Client

Streamerbot → Tab Servers/Clients → WebSocket Clients → Add → Endpoint
Auto Connect ✅Auto Reconnect ✅TLS 1.2 ✅
Kein ws:// selbst tippen — volle URL einfügen (Schema kommt über TLS). Merk dir den Client-Index (erster = 0, die Actions senden an 0).

2 Kanal-Token als persistente Variable

Der Token darf nicht im Klartext in einer Action stehen — er wird einmalig als persistente globale Variable hinterlegt. So in Streamerbot:
  1. Token holen: GW ADMIN → STREAM-VERBINDUNGEN → bei deinem Kanal GENERIEREN → den langen Token kopieren.
  2. Neue Action anlegen (Name egal, z.B. „CC – Set Token") — sie braucht keinen Trigger.
  3. Sub-Action hinzufügen: Core → Global Variable → Set Global Variable. Felder:
    Variablecc_ingest_token
    Valuedein Token aus Schritt 1
    Persisted✅ AN (Pflicht!)
  4. Action einmal ausführen — in Streamerbot Rechtsklick auf die Action → Test Trigger (oder ▶). Der Wert ist jetzt dauerhaft gespeichert; die Action kannst du danach löschen.
Persisted ist zwingend: sonst ist die Variable nach Streamerbot-Neustart weg und CC_IngestConnect findet sie nicht → Kanal bleibt Closed.
